🗂 טבלאות Airtable (שדות חובה)

Content_Ideas

Idea_ID (מפתח), Topic, Goal (Awareness/Promo/UGC), Angle (Before/After, Tips, Offer), Priority (H/M/L), Status (new/approved/rejected)

Content_Queue (מה שמתוכנן לייצור/תזמון)

Queue_ID, Platform (IG Feed/Reels/Stories/Facebook/TikTok), Topic, Caption_Draft, Hashtags, Media_URL(s), CTA, Publish_At (תאריך/שעה), Status (draft/needs-approval/approved/scheduled/published), Owner, UTM (אוטו’), LinkInBio (כן/לא)

Published_Posts

Post_ID, Platform, Publish_At, Caption_Final, Media_URL(s), Permalink, Metrics (JSON: likes, comments, saves, reach, CTR), Campaign

🔁 זרימת עבודה (Happy Path)

טריגר (יומי/ידני): משיכת רעיונות מ-Content_Ideas לפי Status=new/approved.

יצירת דרפט: מחולל GPT בונה כותרת, תיאור, hashtags, CTA, אמוג'יז, וריאציות A/B.

בדיקת מדיה: אם חסר מדיה → מבקש ממך/מהצוות או מייצר קריאת AI ליצירת תמונה/וידאו (אם רלוונטי).

כתיבה ל-Content_Queue: Status=needs-approval. שולח הודעת WhatsApp עם תקציר + לינק לאישור.

אישור: תגובה “מאשר” בווטסאפ או שינוי סטטוס ב-Airtable → approved → תזמון אוטומטי.

תזמון: יוצר פוסט מתוזמן לפי Publish_At (IG/FB/TikTok) + מוסיף UTM.

פרסום: לאחר פרסום, שומר Permalink ומושך מדדים אחרי 24/72 שעות → כותב ל-Published_Posts.

דוח שבועי: WA/מייל עם Top 3 פוסטים, Reach, CTR, שמירות, ואיזו זווית עבדה הכי טוב.

⚙️ Make/n8n – מודולים

Scenario 1: Generate Drafts
Trigger: Manual/CRON → Airtable List Content_Ideas (Status=approved) → OpenAI Create Draft → Upload to Airtable Content_Queue (Status=needs-approval) → Send WA preview.

Scenario 2: Approvals
Trigger: WA inbound “מאשר {Queue_ID}” → Airtable update Status=approved + set Publish_At (אם חסר) → Schedule via Meta/TikTok.

Scenario 3: Scheduler
Trigger: Every 10 min → Airtable filter Status=approved & Publish_At <= now+5m → Publish → Update Status=published + save Permalink.

Scenario 4: Analytics
Trigger: Daily → Fetch Insights (IG/FB/TikTok) for last 3 days → Write Published_Posts.Metrics → If Winner (הכי הרבה Saves/Reach) → Duplicate angle ל-Content_Ideas עם Priority=High.
